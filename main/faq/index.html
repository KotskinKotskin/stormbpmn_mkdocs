    FAQ по Stormbpmn¶

    Этот раздел содержит ответы на частые вопросы, которые возникают у пользователей Stormbpmn.

    Редактор диаграмм¶

    Изменить цвет элемента диаграммы

    Кликните по элементу диаграммы, цвет которого хотите изменить, а затем кликните на кнопку Изменить цвет выделенного в верхней панели управления и выберите желаемый цвет заливки элемента:

    Для изменения цвета заднего фона диаграммы — кликните по фону диаграммы и повторите действия, описанные выше.

    Оставить комментарий к элементу диаграммы

    Кликните по элементу диаграммы, справа вверху от элемента появится кнопка Написать комментарий. Кликните по кнопке и в появившееся поле введите комментарий. Поддерживается многострочный ввод и возможность прикрепить файл:

    Комментарии, оставленные на элементах, дублируются списком виджетов в правой панели управления в разделе Комментарии:

    Виджеты комментариев удобно отражают статус комментария, его важность, принадлежность к элементу на схеме:

    • Естественный цвет виджета комментария и чёрный цвет кнопки свидетельствуют о том, что комментарий активен.
    • Приглушённый (бледный/затенённый) цвет виджета комментария и зелёный цвет кнопки свидетельствуют о том, что комментарий решён.
    • Оранжевый цвет виджета комментария указывает на его важность. Изменить уровень важности комментария можно, кликнув по кнопке в виджете и выбрав из выпадающего списка опцию: Сделать важным / Отменить важность.
    • Быстро перейти к элементу, к которому прикреплён комментарий, можно, кликнув по кнопке в левом верхнем углу виджета комментария.
    Поток управления «в остальных случаях», поток по умолчанию

    По правилам BPMN-нотации при использовании логической развилки один из выходных потоков должен быть отмечен как поток управления «в остальных случаях» (RU-версия). Для этого кликните на нужный выходной поток — откроется меню дополнительных настроек. Кликните по изображению гаечного ключа и выберите пункт: Поток управления «в остальных случаях»:

    видео с примером.

    Подпроцесс по событию

    Подпроцесс по событию в BPMN 2.0 — это тип подпроцесса, запускаемого конкретным событием (сообщением, ошибкой, сигналом или таймером) во время выполнения родительского процесса, с возможностью прерывания или параллельного выполнения для обработки исключений и уведомлений.

    В Stormbpmn Подпроцесс по событию — это опция изменения Развёрнутого подпроцесса. Чтобы создать Подпроцесс по событию, выполните следующие шаги:

    1. Перетащите на диаграмму из левой панели элементов элемент Развёрнутый подпроцесс:

    2. Кликните по элементу Развёрнутый подпроцесс и из выпадающего меню выберите опцию Изменить элемент:

    3. В меню дополнительных опций изменения процесса выберите пункт Подпроцесс по событию:

    Интеграции Stormbpmn с популярными платформами¶

    🏷️ Confluence

    Встроить диаграмму Stormbpmn в документ облачной версии Confluence

    1. Перейдите в окно Редактор диаграмм и нажмите на кнопку Поделиться в верхней панели инструментов:

    2. Активируйте тумблер Доступ по ссылке и кликните по кнопке Встроить iframe:

    В буфер обмена будет скопирован код iframe. Временно его можно вставить в любой текстовый редактор. Позже понадобится только сама ссылка.

    1. Перейдите в облачный редактор Confluence, откройте документ, в который хотите вставить диаграмму, и перейдите в режим редактирования.

    2. Установите курсор в нужную строку документа, кликните по слева от курсора и выберите последний пункт выпадающего списка View more:

    3. Введите в поиск «iframe» и выберите блок Iframe:

    4. В открывшемся окне настроек блока Iframe введите в поля справа следующие значения:

    5. URL — укажите ссылку на iframe из скопированного кода (формата: https://new-test.stormbpmn.com/app/diagram/bb7a69c1-e1a9-4140-8e98-e5aead2cb54b?overlays=...).

    6. Width — задайте 100%.
    7. Height — по желанию.

    Остальные параметры опциональны.

    Работа с файлами и с рабочим пространством Stormbpmn¶

    Перемещение диаграмм между папками

    При создании аккаунта автоматически создаётся команда и рабочее пространство команды с дефолтной папкой Others, в которую по умолчанию сохраняются диаграммы, если не указана другая папка.

    Перемещение диаграммы между папками в разделе Команда¶

    1. Для перемещения диаграммы после её сохранения без указания конкретной папки или для перемещения между папками перейдите в раздел Команда → Модели бизнес-процессов → Папки:

    2. Перейдите в папку, где сейчас находится диаграмма, которую нужно перенести, или в папку Others, если диаграмма была сохранена без указания папки.

    3. Кликните на строку с названием диаграммы (не кликайте по самому названию — это откроет диаграмму в режиме редактирования), затем кликните на кнопку Переместить в верхней панели инструментов:

    4. В открывшемся модальном окне Переместить процессы выберите папку, в которую нужно перенести процессы, и нажмите кнопку Переместить процессы:

    Если нужно переместить несколько процессов — кликайте по строке каждого процесса, который требуется перенести. Выбор по строке работает в режиме мультивыбора.

    Перемещение диаграммы между папками в редакторе диаграмм¶

    1. Откройте диаграмму в редакторе и нажмите на кнопку Сохранить как версию в верхней панели инструментов:

    2. В открывшемся модальном окне Сохранение модели из выпадающего списка Название папки выберите нужную папку и нажмите кнопку Сохранить как версию:

    Оба способа перемещения диаграммы показаны в видео-примере.
